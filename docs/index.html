Introduction to CodeFlow

CodeFlow is a modern, statically-typed programming language designed for building scalable and maintainable applications. It combines the simplicity of Python with the performance of Go, making it an excellent choice for everything from small scripts to large distributed systems.

Why CodeFlow?

CodeFlow was created to address common pain points in modern software development: slow compilation times, complex syntax, and poor error messages. With CodeFlow, you get fast compile times, clean and readable syntax, and helpful error messages that guide you to the solution.

Simple Example

Here's what a simple CodeFlow program looks like:

hello.flow 
// This is a simple CodeFlow program
func main() {
    // Print a message to the console
    print("Hello, World!")
    
    // Variables with type inference
    name := "Alice"
    age := 30
    
    // String interpolation
    print(f"Hello, {name}! You are {age} years old.")
    
    // Conditional statement
    if age >= 18 {
        print("You are an adult.")
    } else {
        print("You are a minor.")
    }
}
Note

CodeFlow uses a minimalistic syntax that reduces boilerplate while maintaining readability. The language is designed to be easy to learn for programmers familiar with C-style languages.

Variables and Data Types

CodeFlow is statically typed, but it includes powerful type inference so you rarely need to explicitly write types. The compiler can infer types from the context in most cases.

Variable Declaration

Variables in CodeFlow can be declared using the := operator for type inference, or with explicit type annotation using the var keyword.

variables.flow 
// Type inference (recommended)
name := "Alice"               // string
age := 30                     // int
height := 5.8                 // float
isStudent := true             // bool

// Explicit type declaration
var score: int = 95
var temperature: float = 72.5
var message: string = "Hello"

// Constants
const PI := 3.14159
const APP_NAME := "MyApp"

// Multiple assignment
x, y := 10, 20
a, b, c := "foo", 42, false

// Type annotations for function parameters
func add(a: int, b: int) -> int {
    return a + b
}
Data Type Description Example
int Integer numbers 42, -7, 0
float Floating point numbers 3.14, -0.001
string Text data "Hello", 'World'
bool Boolean values true, false
array[T] Fixed-size collection [1, 2, 3]
list[T] Dynamic-size collection list[int]{1, 2, 3}
map[K]V Key-value mapping map[string]int{"age": 30}
Tip

Use type inference (:=) for local variables to keep your code concise. Use explicit type annotations for function parameters and return types to improve readability and catch type errors early.

Control Flow

CodeFlow provides familiar control flow constructs with some enhancements for safety and expressiveness.

Control Flow Examples

CodeFlow supports if/else statements, loops, and pattern matching for complex conditional logic.

control.flow 
// If-else statement
func checkNumber(n: int) {
    if n > 0 {
        print("Positive")
    } else if n < 0 {
        print("Negative")
    } else {
        print("Zero")
    }
}

// For loop (like a while loop)
i := 0
for i < 5 {
    print(i)
    i += 1
}

// For loop with range
for i in range(0, 10) {
    print(i * 2)
}

// For-each loop
names := ["Alice", "Bob", "Charlie"]
for name in names {
    print(f"Hello, {name}!")
}

// Pattern matching (switch with superpowers)
func describe(x: any) {
    match x {
        case 0:
            print("Zero")
        case s: string if len(s) > 5:
            print(f"Long string: {s}")
        case n: int:
            print(f"Number: {n}")
        case _:
            print("Something else")
    }
}

// Error handling with try/catch
func riskyOperation() {
    try {
        result := potentiallyFailingFunction()
        print(f"Success: {result}")
    } catch err: IOError {
        print(f"IO Error: {err.message}")
    } catch err: ValueError {
        print(f"Value Error: {err.message}")
    } catch err {
        print(f"Unexpected error: {err}")
    }
}
Warning

Unlike some languages, CodeFlow does not have traditional C-style for loops (for (i=0; i<n; i++)). Use for i in range(start, end) instead, which is safer and less error-prone.

Functions

Functions are first-class citizens in CodeFlow. They can be assigned to variables, passed as arguments, and returned from other functions.

functions.flow 
// Basic function with parameters and return type
func add(a: int, b: int) -> int {
    return a + b
}

// Function with multiple return values
func divide(a: float, b: float) -> (float, bool) {
    if b == 0 {
        return 0, false  // Division by zero
    }
    return a / b, true   // Success
}

// Function with default parameters
func greet(name: string, greeting: string = "Hello") {
    print(f"{greeting}, {name}!")
}

// Variadic function (accepts any number of arguments)
func sum(numbers: ...int) -> int {
    total := 0
    for n in numbers {
        total += n
    }
    return total
}

// Higher-order function (takes function as parameter)
func applyTwice(f: func(int) -> int, x: int) -> int {
    return f(f(x))
}

// Anonymous function (lambda)
square := func(x: int) -> int {
    return x * x
}

// Using the functions
result := add(5, 3)                // 8
value, ok := divide(10.0, 2.0)    // 5.0, true
greet("Alice")                     // "Hello, Alice!"
greet("Bob", "Hi")                 // "Hi, Bob!"
total := sum(1, 2, 3, 4, 5)       // 15
final := applyTwice(square, 2)    // 16 (square(square(2)))
Function Signatures

Every function in CodeFlow has a signature that defines its parameter types and return type. The compiler uses these signatures to ensure type safety across your entire program. Function signatures are also used for function overloading and interface implementation.
